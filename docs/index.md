# 自己 PR
## 強み
### 設計〜実装を通して価値を届ける実行力
課題に対して「何のために作るか」「どう実現するか」を軸に設計・実装を進め、仕様通りではなく価値ある形を目指して工夫を重ねてきました。PdMやチームと密に連携しながら精度の高い仕様を構築し、安定した品質とスピード感を両立して着実にリリースまでやり切る力が評価されています。

### 自発的な改善行動とチーム志向のリーダーシップ
役職に関係なく、振り返りのファシリテーションや相談しやすい雰囲気づくりなど、チーム全体の改善と成果の最大化に取り組んできました。メンバーの特性に合わせたミーティング運営や、主体的な課題発見・提案を通じて、協調を重視したリーダーシップを発揮しています。

### 他職種と協力しながら開発を前に進める調整力
PdM・デザイナー・QAなどと連携し、仕様検討・見積もり・調整を円滑に進行。難易度の高い変更に対しても影響やリスクを整理し、合意形成に努めてきました。相手に合わせた丁寧な説明と支援により、開発を止めずに推進する調整力を発揮しています。

### 信頼される存在としての柔軟なサポート力
問い合わせ対応や進捗のフォロー、仕様理解の支援など、必要に応じて自発的に動き、周囲からの信頼を築いてきました。役割にとらわれず、開発以外の業務も丁寧に対応することで、チーム全体がスムーズに動けるよう柔軟にサポートしてきました。

## 大事にしている考え
### チームで成果を出すことを最優先にする
自分個人の成果よりも、チーム全体としてのアウトカムを重視しています。役割に関係なく、必要に応じてサポートに回ったり、改善提案を行ったりする姿勢を大切にしています。

### 価値ある形でプロダクトを届ける
「どう作るか」だけでなく「なぜ作るのか」に目を向け、ユーザーや運用者の立場に立って設計・実装を行うことを心がけています。

### 丁寧かつ前向きなコミュニケーションを意識する
相手にとってわかりやすく・話しやすい雰囲気を作ることを意識しています。意見がぶつかる場面でも、落ち着いて柔らかい態度で対話を進めるよう努めています。

## やりたいこと
### プレイヤーとして開発に関わる
現在はキャリアの中でも特に実装力を高めたいと考えており、手を動かしながらチームやプロダクトに貢献することを重視しています。

### スクラムなどのアジャイルな開発プロセスでのチーム開発
振り返りや計画を通じてチームで改善を重ねながら、継続的に価値を届けるスタイルが得意です。

### Webアプリケーションのバックエンド・フロントエンド両方に関わる開発
リリースする機能に対して自分で責任を持つ姿勢を大切にしており、そのためにバックエンド・フロントエンドをまたいで開発できることにやりがいを感じます。ただし、チームとして責任感を共有しながら開発できる環境であれば、自身がすべてを担うことにこだわっているわけではありません。

### (中長期)マネジメント方面のキャリア
中長期のキャリアとしては、マネジメント方面に進みたいと考えています。ただし、マネジメントという大枠の中で具体的にどういう事をしたいか、という事はまだ固まりきっておらず、この先模索していきたいと思っています。
